### GRPO 中的优势值（Advantage）计算

在 GRPO（组相对策略优化）强化学习阶段，模型不再依赖复杂的外部价值模型，而是通过对比同一组内多个回答的相对表现来进行自我进化。第 $i$个回答的**优势值** $A_i$ 计算公式如下：
$$
A_i=\frac{r_i-\operatorname{mean}\left(r_1, r_2, \ldots, r_G\right)}{\operatorname{std}\left(r_1, r_2, \ldots, r_G\right)}
$$

1. $A_i$ 衡量了第 $i$ 个回答相对于组内平均表现的“优越程度”。它是模型更新参数的指挥棒。如果 $A_i$ 为正，模型会增加生成类似逻辑路径的概率；如果为负，则会降低其概率，它是实现“优中选优”的核心；
2. $r_i$ 是原始奖励分数，由预设的奖励函数（如格式检查、反馈逻辑检查、信息增益等）直接为第 $i$ 个回答打出的原始分值。它是评估模型表现的**直接依据**，反映了该回答在格式、逻辑和策略上的绝对质量；
3. $G$ 为组大小，是模型针对同一个提示词一次性生成的回答总数（在本项目中设置为 16）。较大的组大小提供了更丰富的对比样本，能够让计算出的平均值和标准差更具代表性，从而使训练更加**稳健**；
4. $mean(r_1,r_2,…,r_G)$ 是组内奖励平均值，即这一组内所有 16 个回答得分的数学平均数。它作为一个**动态基准线**，通过减去平均值，算法将绝对的分数转化为了相对的“胜负信号”。这意味着即便在非常困难的任务中（全组得分都较低），表现相对较好的回答依然能获得正向优势，引导模型持续学习；
5. $std(r_1,r_2,…,r_G)$ 是组内奖励标准差，是衡量这一组回答得分的离散程度（波动大小）。它是归一化的核心。通过除以标准差，优势值会被限制在一定的量级范围内（通常在 -1 到 1 附近波动）。这能防止因为奖励分数跨度过大导致模型权重更新过猛（梯度爆炸），确保了训练过程的**平滑与稳定**。

计算 Advantage 是为了将原始奖励分转化为**相对竞争信号**。它通过减去组内平均得分，将奖励信号转化为“优于平均”或“低于平均”的明确指令，从而解决了单纯依靠原始分导致的训练不稳定问题。这使得模型即便在面临高难度残局（全组得分较低）时，依然能通过识别并强化“相对较优”的逻辑路径实现持续进化，最终确保模型朝着信息熵最大化的博弈策略稳健收敛。
